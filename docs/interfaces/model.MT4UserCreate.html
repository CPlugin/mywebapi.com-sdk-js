Same writable fields as CPlugin.SaaSWebApps.WebAPI.DTOs.MT4.v2.MT4UserUpdate minus the explicit `Balance`/`Credit` (those should come through dedicated balance operations after the account exists). The wrapper allocates the next free login id when `Login = 0`; clients may also request a specific id by setting `Login > 0` (the server rejects collisions). Password / OTP / API-data fields are NOT on this DTO. After successful creation, set the initial password via a separate `POST UserPasswordSet/{login}` call.
