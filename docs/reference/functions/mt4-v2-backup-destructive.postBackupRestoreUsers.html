Destructive — requires `?confirm=true`. Manager (live) call to the wrapper's `BackupRestoreUsers(UserRecord[] users)`. Each input `MT4UserRestoreInput` is mapped to a fresh `UserRecord` with the narrow restore field set — secrets, OTP, server-managed timestamps, and reserved blobs are NOT carried (see DTO docs).
Idempotency-Key header is strongly recommended: a network blip during a multi-user restore can leave the client uncertain whether the write happened. Without the key, retry will double-write.
