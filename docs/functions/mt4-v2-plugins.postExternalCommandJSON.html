Manager (live) call wrapping `ExternalCommandJSON`. The MT4 server forwards the body to whichever installed plugin claims the command first; the first plugin returning `RET_OK` wins and its response becomes the v2 payload. The request body is sent verbatim (no field renaming, no schema enforcement) so the plugin author owns the over-the-wire contract on both ends. The wrapper trio (`ExternalCommand` for binary marshal, `ExternalCommandCustom` for caller-supplied serializer) is intentionally not exposed in v2 — those variants require compile-time struct layouts shared between client and plugin, which a REST surface cannot guarantee. Plugin developers who need binary transport should keep using the wrapper directly from the WebAPI process or build a dedicated binary endpoint.
